# Database Terminology

## **Level 1: Core Concepts & Data Foundation**

*Before touching a database, one must understand what data is and the systems that manage it.*

* Information structure (Data, Information, Knowledge, Wisdom)
* Donnée persistante
* Donnée transitoire
* Base de données
* DBMS / SGBD (Système de gestion de bases de données)
* Métadonnée
* Data dictionary
* Data catalog
* Redondance d'information

## **Level 2: Database Architecture & Lifecycle**

*Understanding the stages of building a database and its multi-tiered architectural abstraction.*

* Database Lifecycle (Analysis, Modeling, Implementation, Test/Evaluation, Use)
* Couche externe / External level
* Couche conceptuelle
* Couche logique / Schéma de communauté (Community Schema)
* Couche physique / Couche interne (Internal Schema)
* Indépendance des données

## **Level 3: Conceptual Modeling & Entity-Relationship**

*Planning the database structure using high-level conceptual blueprints before writing any code.*

* Modélisation de données
* Modèle entité-association
* Entity / Entité
* Entity Class / Classe d'Entité (Faible, Forte)
* Attribut (Atomique, Composite, Dérivé, Identifiant)
* Association (Binaire, Ternaire, Récursive, Cyclique)
* Roles in Association
* Association Cardinality (One-to-One, One-to-Many, Many-to-Many)
* Cardinalité (Totale, Partielle, Exclusive, Overlapping)

## **Level 4: Advanced Modeling & Object-Oriented Concepts**

*Handling complex relationships, hierarchies, and object-oriented integrations.*

* Modèle orienté objets
* UML diagramme de classes
* Relation is-a
* Spécialisation / Généralisation
* Multiple Specialization
* Héritage / Héritage multiple

## **Level 5: The Relational Model & Data Integrity**

*Translating conceptual models into relational tables and enforcing strict data rules.*

* Modèle relationnel
* Schéma relationnel
* Clé primaire (PK)
* Clé étrangère (FK)
* Valeur NULL
* Contrainte d'intégrité (Primary Key, Foreign Key, Domain, Line, Column)
* Intégrité référentielle
* Vue / View

## **Level 6: Querying & Languages**

*Communicating with the database to define structures and manipulate information.*

* Langage déclaratif
* Langage procédural
* SQL (Structured Query Language)
* DDL (Data Definition Language)
* DML (Data Manipulation Language)
* DRL (Data Retrieval Language)

## **Level 7: Transactions & Concurrency**

*Ensuring data safety, accuracy, and performance in multi-user, real-time environments.*

* Transaction
* ACID (Atomicité, Consistance, Isolation, Durabilité)
* Accès concurrents
* Roll back

## **Level 8: History & Tooling**

*Legacy frameworks and modeling software used in the domain.*

* CODASYL
* DB-Main
